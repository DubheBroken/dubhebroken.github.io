注意事项

使用指令的时候，第一个参数”以前的目录”必须是不存在的，否则会报错。

第二个参数不会检验是否存在，如果不存在指令不会报错，且软件运行时也不会报错。

所以一定要确认第二个参数的目录存在，否则可能导致软件发生异常。

高性能安卓模拟器 Genymotion安装和配置

2019-12-27 Android

由于Genymotion的官网时常改版，本文具有一定的时效性。仅供参考，切勿生搬硬套。

Genymotion官网

点开上面的蓝字链接，找到点击顶部的 Desktop Sign In

创建账号

点击登录页中的 Create an account 新建一个账号

参考下图填写表单

然后去邮箱里面收邮件激活账号

然后再回到刚才的登录页，输入账号密码点 Sign in

登录以后就可以看到下载按钮了

有自带 V-box 的和不带 V-box 的两个版本，如果你的电脑上有 V-box 就选第二个，否则选第一个。

配置Genymotion

安装步骤无脑下一步略过不讲。

登录Genymotion客户端

登录刚刚注册的 Genymotion 账户

输入完点右下角 Next，然后你会看到下面这个界面，找你要许可证，选第二项个人使用。

然后是霸王条款，读不读都TM得同意

选项设置

然后终于可以看到这东西的主界面了开不开森٩(๑>◡<๑)۶

先别急着开始new模拟器，先改改设置，点击左上角菜单栏的 Genymotion → Settings

其中 Network 就是刚刚登录页见过的代理设置，V-box 那栏建议不要动他。

重点是这个 ADB 设置

ADB 全称 Android Debug Bridge 这个工具可以使用指令控制Android设备，debug时必须使用这个工具连接。如果 ADB 路径不同，Android Studio 将无法连接到 Genymotion 创建的模拟器。

所以这里要选第二项，目标选 Android Studio 的 Android SDK 路径。

是 SDK 的路径，不是选择 adb.exe 所在的位置。

顺便提一下不同平台的 SDK 不通用，也就是 Windows、MacOS、Linux 的 Android SDK 彼此不通用。

Misc中可以选择截图保存的位置，以及清除缓存

新建模拟器

找一个看着顺眼的镜像点击右边的三个点，选择 Install

然后会弹出这么一个框框

填好以后点击右下角的 INSTALL，如果你不知道怎么填，全默认就好了。

然后在顶部的 My installed devices 中可以看到创建的模拟器在跑条，如果下载进度太慢的话可以考虑开个代理或者使用下载工具下载镜像。

使用下载工具下载镜像

如果下载镜像太慢而你又没有梯子，那你可以考虑使用下载工具来下载镜像。

打开 C:\Users\[你的用户名]\AppData\Local\Genymobile 这个目录

（如果看不见 AppData 请先在文件夹选项里打开显示隐藏文件夹）

打开 genymotion.log 这个文件，拉到底，有一个类似

Starting download of "https://dl.genymotion.com/dists/9.0/ova/genymotion_vbox86p_9.0_190715_123003.ova"

这样的日志

复制双引号里的这个 .ova 结尾的地址，扔进下载工具里。

下载完之后回到 Genymotion 中，点叉取消创建模拟器。

然后打开 C:\Users\[你的用户名]\AppData\Local\Genymobile\Genymotion\ova 这个目录，里面有一个下载到一半的镜像，删了它，把你下载好的镜像扔进去。

再重新创建模拟器，会跳过下载直接开始安装。

镜像是跟随安卓版本的，同系统版本的镜像只用下载一次，删除模拟器不会删除镜像，下次创建时可直接使用。

编辑模拟器

点击创建好的模拟器右边的三个点展开操作菜单

编辑 就是修改创建时填写的模拟器参数，只能在关机状态下操作。只能修改部分参数，

删除 只能在关机下操作，删掉这个模拟器以及本地的这个模拟器的文件，不会删除镜像。

克隆 只能在关机下操作，以模拟器当前的状态创建一个一毛一样的克隆，创建的时候要给克隆体起个名字。

设置模拟器

双击开启模拟器，Genymotion 的模拟器是给予 x86 架构的，启动速度非常快。

启动以后默认的时区是0区，语言是英文，可能会影响的APP正常运行。所以要先调整这两个选项。
